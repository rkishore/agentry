## Sprint goal

Stand up an end-to-end skeleton in which evals, observability, and audit logging are
load-bearing from the first commit, proven by a single end-to-end run — on local fixtures
and fakes alone — that ingests, retrieves, answers, scores, writes a typed run-event log,
and emits a trace span.

## Pragmatic decisions

- **Language: Python; package name: `agentry`.** Both confirmed by the human.
- **License: MIT.** Frictionless for a personal open-source learning repo with no commercial aim.
- **Hexagonal layering (core / application / infrastructure / app),** ports-and-adapters style.
  Makes the later Chroma → pgvector → Pinecone and hand-built → framework swaps config changes
  rather than rewrites.
- **`core` has zero third-party dependencies** (stdlib dataclasses + `typing.Protocol`). Keeps the
  domain free of infrastructure concerns and trivially testable.
- **Domain- and benchmark-specific code stays at the edges** (FinanceBench loaders, finance
  graders, future SOP tools live in `app`/`infrastructure`, never `core` or `application`).
  This is what keeps the "swap in your own corpus" reusability story true.
- **Audit is a typed event stream from day one,** not a single event type. Phase 0 ships five
  event types; the taxonomy grows (step/tool/approval events) in the agentic phase. Rationale:
  trace/event analysis is the feedback loop that drives every later improvement.
- **Phase-0 implementations are deliberately thin:** fixed-size chunking, top-k dense retrieval,
  a single LLM call, normalized-match grading. Each gets upgraded later behind the same port.
- **Local `SentenceTransformerEmbedder` (384-dim) as the real embedder** (on-prem ethos, no key).
  Tests use `FakeEmbedder` so CI is fast and hermetic.
- **Two real LLM adapters behind the `LlmClient` port: `OpenAiLlmClient` and `AnthropicLlmClient`.**
  The composition root selects one via `AGENTRY_LLM_PROVIDER` (`openai` | `anthropic`), defaulting
  to `openai`. Having two adapters behind one port is itself a check that the port is honest. The
  human sets `OPENAI_API_KEY` / `ANTHROPIC_API_KEY` in their own environment; keys are never
  hardcoded or committed. Tests use `FakeLlmClient`, so CI needs no key and makes no network calls.
- **Grading is deterministic this sprint** (no LLM judge). Structured output extraction and
  Ragas/numeric-tolerance/judge grading are Phase 1; the optional `Answer.fields` slot is the seam.
- **Tooling: `uv`** for install/lockfile and `uv run` for gate commands (pip is a fallback).
- **No copyright-header gate** on source files; the MIT `LICENSE` at the repo root covers licensing.
- **Python naming** drops the C# `I`-prefix on ports (`Embedder`, not `IEmbedder`); infra adapters
  keep a tech prefix (`Chroma…`, `SentenceTransformer…`, `OpenAi…`, `Anthropic…`).
- **Test naming:** `MethodOrScenario_Condition_ExpectedResult`.

## Evaluator success criteria

All must be true. A single failure fails the sprint.

- **Gate 1 — Static.** `make install` succeeds; `make lint` (ruff) reports zero errors;
  `make typecheck` (mypy strict) exits 0 with zero errors.
- **Gate 2 — Tests.** `make test` (pytest) exits 0, all green, no skipped tests.
- **Gate 3 — Architecture boundaries.** `make boundaries` (import-linter) passes. `agentry.core`
  imports no third-party package and nothing from `application`/`infrastructure`/`app`;
  `agentry.application` imports only `agentry.core`; `agentry.infrastructure` imports only
  `agentry.core` plus its own tech libs; `agentry.app` may import all. Any violation is an
  automatic fail even if the code compiles.
- **Gate 4 — Deliverables present.** Every file path and every type above exists. Verify with
  `find` and `grep`, not memory.
- **Gate 5 — No scope creep.** No files or types outside this contract. Flag extras in the report;
  auto-fail only if they violate boundaries or introduce breaking changes.
- **Gate 6 — Vertical slice.** `test_slice_e2e.py` passes and asserts, in one run:
  (a) a numeric `EvalResult` score over the fixture `eval.json`;
  (b) the `JsonlAuditSink` file contains an append-only typed event stream for the run —
  at minimum `RunStarted`, `RetrievalCompleted`, `LlmCallCompleted`, `AnswerProduced`,
  `EvalCompleted` — collectively carrying prompt, retrieved chunk IDs, model id, tokens,
  latency, output, and eval verdict;
  (c) at least one OTel span was emitted (in-memory exporter).
  `README.md` documents the manual `make slice` run against the real embedder + LLM.
